# 智能图片方向检测与自适应显示

## 🎯 问题背景

在正方形网格布局中，不同方向的图片需要不同的处理策略：

### 旧方案（统一 contain）
```
横拍图 (16:9): ✅ 上下留白，完整显示
竖拍图 (9:16): ❌ 上下大量留白（浪费空间）
正方形 (1:1):  ✅ 完美填满
```

---

## ✨ 解决方案：智能方向检测

### 核心思路

根据图片的**宽高比**自动选择最优显示策略：

| 图片类型 | 宽高比 | 判定条件 | 显示策略 | 效果 |
|---------|-------|---------|---------|------|
| **横拍图** | > 1.1 | `width/height > 1.1` | **contain** | 上下留白，完整显示 |
| **竖拍图** | ≤ 1.1 | `width/height ≤ 1.1` | **cover** | 左右可能裁剪，填满正方形 |
| **正方形** | ≈ 1.0 | `width/height ≈ 1` | **cover** | 完美填满 |

## 🔧 技术实现

### 1. **前端 NImage 组件（小图片）**

```typescript
const handleImageLoad = (event: Event) => {
  const img = event.target as HTMLImageElement
  
  if (img.naturalWidth && img.naturalHeight) {
    const aspectRatio = img.naturalWidth / img.naturalHeight
    
    if (aspectRatio > 1.1) {
      // 横拍图：contain 模式
      imageFit.value = 'contain'
    } else {
      // 竖拍图或正方形：cover 模式
      imageFit.value = 'cover'
    }
  }
}
```

**关键点：**
- 监听 `@load` 事件获取图片原始尺寸
- 使用 `naturalWidth/naturalHeight` 计算真实宽高比
- 动态绑定 `:object-fit="imageFit"`

---

### 2. **Canvas 缩略图生成（大图片）**

```typescript
const generateThumbnail = async (canvas: HTMLCanvasElement, src: string) => {
  const img = new Image()
  await load(img, src)
  
  const aspectRatio = img.width / img.height
  
  if (aspectRatio > 1.1) {
    // 横拍图：contain 模式
    drawWidth = maxWidth
    drawHeight = img.height * (maxWidth / img.width)
    drawX = 0
    drawY = (maxHeight - drawHeight) / 2  // 垂直居中
  } else {
    // 竖拍图或正方形：cover 模式
    const scale = Math.max(maxWidth / img.width, maxHeight / img.height)
    drawWidth = img.width * scale
    drawHeight = img.height * scale
    drawX = (maxWidth - drawWidth) / 2  // 水平居中
    drawY = (maxHeight - drawHeight) / 2
  }
  
  ctx.drawImage(img, drawX, drawY, drawWidth, drawHeight)
}
```

## 💡 设计哲学

### 1. **内容优先**
- 横拍图通常包含更多横向信息（风景、建筑）
- 使用 contain 保护完整性，避免裁剪重要内容

### 2. **视觉统一**
- 竖拍图（人像、树木）主体通常在中间
- 使用 cover 轻微裁剪不影响主体，且保持布局统一

### 3. **智能决策**
- 无需用户手动设置
- 自动检测 + 最优策略
- 兼顾美观与实用性

## 🔍 阈值说明

### 为什么选择 1.1 作为分界线？

```
常见图片格式宽高比：
- 16:9  = 1.78  → contain（横拍）
- 4:3   = 1.33  → contain（横拍）
- 3:2   = 1.50  → contain（横拍）
- 1:1   = 1.00  → cover（正方）
- 9:16  = 0.56  → cover（竖拍）
- 3:4   = 0.75  → cover（竖拍）
```

**1.1 阈值**可以正确分类所有常见格式，并留出 10% 的容错空间。
